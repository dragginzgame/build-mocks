Collection — Sale defaults.

The collection sub-page where a creator sets the default secondary-sale policy every token in the collection inherits at go-live. Like claim defaults it is tier-indexed (a default per rarity tier), auto-saves, and is required-valid before Review and frozen at Live — a token cannot go Live without a locked sale policy. In V1 each tier holds one thing: the economic split (project fee + royalties). Restriction states & per-copy limits are V2 (per token). Mirrors claim-defaults-states-gallery. Source: Design Documentation/sale-rules-design-wip.md.

Sibling of Claim defaults. Related: Rarity · Revenue presets · Marketplace
V1 scope: sale defaults carry only the fee distribution (per-tier project fee + royalties). The restriction states (Tradeable / Gift-only / Sale-only / Soulbound) and per-copy sale / transfer limits are V2 — authored per token, gated by the collection’s sale / transfer guards. Part A note (not UI) — what isn't here

Sale defaults are the token-frozen half of a sale. Two things deliberately live elsewhere: the marketplace / referrer fee is the facilitator's fee (on the Toko marketplace it goes to Toko — that's how Toko monetises secondary, instead of a token-frozen cut), set at the listing/marketplace layer and deducted at settlement on top of this split; and the listing currency is the seller's choice at list time. Neither is durable per-collection policy, so neither belongs in a collection-wide default. What is here — the economic split and restrictions — travels with every copy after Live.

Part B note (not UI)

Sale defaults follow the same lifecycle as the collection's other core policy (Attributes, Supply, Rarity, Guards, Claim defaults): editable in Draft and Review, required-valid before Review, locked at Live. Because collection/Guardian policy is immutable after Live, the policy is authored while Draft/Review and snapshotted at Review → Live as locked_sale_policy; a token cannot go Live without it. A standing default (Tradeable, no limits, default split) is applied so even pre-marketplace Live tokens already carry rules and aren't stranded. Go-Live/Review validation is surfaced on the collection's status checklist (see collection-states-gallery); this sub-page shows field-level state inline only.

Part C note (not UI)

In V1, a tier carries one thing: the split (economic, frozen) — a project fee up to 2% plus royalties — ≤ 3% total — across any number of Live beneficiaries (the creator must be one), each Forever or on a frozen schedule evaluated at resale. Total frozen deduction ≤ 5%, so the seller keeps ≥ 95% before the referrer fee, and Toko takes no token-frozen cut on secondary. The only go-Live gate is a valid split on every tier. Restriction states (Tradeable / Gift-only / Sale-only / Soulbound) and per-copy sale / transfer limits are V2 — authored per token, gated by the collection’s sale / transfer guards; they are not a V1 collection sale default. The marketplace / referrer fee and listing currency are not part of this (Part A note).
