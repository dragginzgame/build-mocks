Marketplace token (buy) — states gallery

The buyer-facing token view on the Marketplace — a quick-view modal that opens over the browse grid (which stays open behind it), so closing it returns you to your place. Only listed tokens open this view. It mirrors the launchpad claim view — same viewer, same side panel, same confirm-before-settle modal and result — so buying feels like claiming. The difference is the panel: a fixed-price listing (price, floor, last sale) instead of a vendor drop. v1 is Buy Now only — no bidding, no offers. Source: Design Documentation/secondary-marketplace-design.md.

01Listed & buyable — token quick-view (modal)Opened by clicking a listed token in the marketplace grid: a quick-view modal over the browse, which stays open behind it — closing returns you to your place in the grid (so there is no "More from this collection" rail). Left: the viewer with identity, attributes and market history. Right: the buy panel — seller, price, collection floor and last sale, listing expiry, and the primary Buy now CTA. The seller set a referral fee (1–10%) when listing; a higher fee places the listing nearer the top of the marketplace. Only listed tokens open this view.

02Buy confirm — wallet & payment contextBuying is explicit and user-confirmed, identical in shape to the claim confirm. Before anything settles the modal restates what's bought and what's paid from the connected wallet, including the running balance. The buyer pays the list price only — there is no separate network/platform fee added to the buyer. On confirm, settlement is atomic and ownership transfers to the buyer.
