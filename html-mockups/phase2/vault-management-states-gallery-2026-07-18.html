Vault management · own sidebar area

Vault management · states gallery

The vaultie side of the V1 vault system, per the 2026-07-18 respec — reworked 2026-07-20 to Donna's review: the vault nav bar appears on every in-vault page (with an "All vaults" item at the start and Profile / Access at the end, Overseers only — Dwellers get a reduced bar); vaults can be pinned exactly like collections (pinned vaults appear in the app sidebar under Vault management, mirroring "Pinned collections" — that sidebar pinning still exists); the profile is full-width sections matching the collection/user profile layout with the standard Save; Access matches the project Access page and adds a permissions matrix; tables carry the standard white list bar; the item view matches the marketplace/launchpad token view; and Inventory uses the minted-token card grid with bulk select. Vault management is its own top-level area, gated per account; access is granted per vault location.

Source: Design Documentation/Phase2/vault-provider-system-design.md. Templates: Access (team + add-button placement) · Minted (card grid, select, list bar) · Marketplace token (public token view) · user-profile (address entry, Save changes).
