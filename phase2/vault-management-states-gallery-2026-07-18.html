The vaultie side of the V1 vault system, per the 2026-07-18 respec: vault management is its own top-level area with its own sidebar entry, separate from the studio, gated per account. V1 serves Toko-owned RWA collections only. Access is granted per vault location. Supersedes vault-workspace-states-gallery-2026-07-17.html.

Source: Design Documentation/WIP/vault-provider-system-design.md → V1 respec (2026-07-18). Templates: Access (team) · Token draft (shipment detail) · user-profile (address entry).

04Vault Access — the team, per locationThe access-gallery pattern, vault-scoped. Access is per vault location, not for all vaults — a person working both facilities is added to each. 10Manage Inventory — Vaulted · Redeem Requested · Return RequestedEverything currently at this location, in three tabs. Vaulted: the only action is updating an item's location. Redeem Requested: retrieve → package → label → shipping area, one item at a time, entering shipping company, tracking and estimated arrival; if the same user has other redeem-requested items here, a hint offers combining them into one parcel. Return Requested: the collection asked for items back — bulk-select and apply one set of shipping details to all of them.
