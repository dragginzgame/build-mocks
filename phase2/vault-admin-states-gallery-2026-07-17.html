Vault admin (registry) · states gallery

The Toko-internal registry where a platform admin adds and manages vault providers. Follows the admin-creator-access pattern (account lookup, resolved identity read-only). Registry entries carry the platform stage lifecycle — Draft → Review → Live — and only Live providers appear in the creator-facing vault directory. Suspension ends a live relationship without deleting history.

Source: Design Documentation/WIP/vault-provider-system-design.md §2 (registry; lifecycle updated 2026-07-17 to Draft/Review/Live). Pattern: Admin · creator access. Related: Vault workspace (vaulties).

04Lifecycle — Draft → Review → Live (+ Suspended)The platform stage pattern, applied to registry entries. Draft: Toko is still filling the record. Review: vetting in progress — checks must pass before Live. Live: shown to creators in the directory, and the provider's account gets the vault workspace. Suspended ends a live relationship: hidden from creators, workspace frozen read-only, history preserved forever.
Stage rail
Draft→ Review→ Live· Suspended

Review checklist (all must pass): every required field present · primary contact's Toko account resolved. (Services, products accepted and ships-to are maintained by the vault itself, on its Vault Management profile — not part of this record.) Review → Draft is always available for corrections, mirroring the collection pattern. Only Live entries appear in the vault directory or are selectable by an RWA collection.
